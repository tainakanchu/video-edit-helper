# video-edit-helper 仕様書

旅行 VLOG など長時間撮りためた動画素材を俯瞰・確認し、「使う部分」をメモ・選定して、DaVinci Resolve などの編集ソフトにラフカットとして受け渡すための映像編集補助ツール。

特定の旅行に特化しない汎用ツールとして開発するが、設計判断の基準として「台湾一周旅行の VLOG 素材(約 80〜90 時間 / 13 日分)」をリファレンスユースケースに採用している。本文中の具体的な数値・機材はこのユースケースに基づく例である。

## 1. 背景と解決したい課題

- 数十時間規模の素材(リファレンスでは 1 日 6〜7 時間 × 13 日分)を前に、どこを使えばいいか分からず編集が止まりがち
- 素材を全部再生して確認するのは現実的でない → **再生せずに中身を把握できる俯瞰手段**が必要
- 「ここ良いかも」と思った箇所を記録する場所がない → **タイムコードに紐づくメモ**が必要
- 選んだ箇所を編集ソフトに持っていく作業が手間 → **DaVinci Resolve へのラフカット自動生成**で解消
- 80 時間を数週間かけて消化する**持久戦**になる → 「どこまで見たか」の進捗管理と、倍速・スキップ等のレビュー効率装備が必須

## 2. リファレンスユースケースと動作環境

| 項目 | 内容(リファレンスユースケースの例) |
|---|---|
| メインカメラ | アクションカム / ジンバルカメラ(例: Feiyu Pocket 2S。長回し動画は約 3GB ごとにファイル分割される) |
| サブカメラ | スマートフォン(設定により H.264 / HEVC / 10-bit が混在しうる。**「ここぞ」の瞬間に撮るのでハイライトが偏在しがち**) |
| 素材置き場 | 外付け SSD / HDD |
| 最終編集ソフト | DaVinci Resolve |
| 実行環境 | **Windows ネイティブ**(Node.js for Windows + ffmpeg / whisper.cpp の Windows バイナリ) |
| 開発環境 | WSL2(ツール類は nix で管理) |
| GPU | なし / 弱め → 重い解析は CPU で夜間バッチ実行する運用 |
| 発話言語 | 多言語混在を想定(リファレンスでは日本語メイン + 現地語(中国語)) |

## 3. アーキテクチャ方針

**ローカル Web アプリとして開発し、将来 Tauri で包んでデスクトップアプリ化できる構成にする。**

```
┌─ ブラウザ (UI) ──────────────────┐
│  Vite + React + TypeScript       │
└────────────┬─────────────────────┘
             │ HTTP / JSON API(疎結合を維持)
┌────────────┴─────────────────────┐
│  ローカルサーバー (Node.js)       │
│  - 素材スキャン / メタデータ抽出  │
│  - ジョブキュー(夜間バッチ)     │
│  - メモ / 選定データの永続化      │
└──┬───────────┬──────────┬────────┘
   │ 子プロセス │          │
 ffmpeg     whisper.cpp  Silero VAD
 (サムネ/プロキシ) (文字起こし)  (発話区間検出)
```

- pnpm workspace モノレポ
  - `packages/web` — フロントエンド(Vite + React)
  - `packages/server` — ローカルサーバー(Node.js)
  - `packages/shared` — 型定義・API スキーマ共有
- フロント⇔サーバーは HTTP/JSON のみで通信し、Tauri 移行時もサーバー部をサイドカーとして再利用できるようにする
- Windows ネイティブ実行を最優先とし、パス処理は Windows パス前提で実装(WSL 開発時も動くようにする)

## 4. ドメインモデル

```
Trip(撮影プロジェクト全体)
 └─ Day(日別プロジェクト × 13)── 編集単位。UI のトップナビゲーション
     └─ Clip(論理クリップ)──── 連番分割ファイルを 1 本にまとめた単位
         ├─ SourceFile[](実ファイル。3GB 分割された MP4 群)
         ├─ reviewStatus ──────── 未確認 / 確認中 / 確認済み
         ├─ watchedRanges[] ───── 再生済み区間(自動トラッキング)
         ├─ Note[](付箋メモ:1 点のタイムスタンプ + テキスト + タグ + ステータス)
         └─ Selection[](選定範囲:イン点/アウト点 + メモ + タグ + 評価)
 └─ RoughCut(Day ごと)──── Selection を並べた書き出し用シーケンス
```

- **連番グルーピング**: ファイル名の連番・撮影日時の連続性から分割ファイルを自動検出し、1 本の論理クリップとして連続再生・連続タイムコードで扱う(アクションカム等の 3〜4GB ファイル分割対策の中核機能)
- **タイムコード**: Clip 内は「結合後の通しタイムコード」で扱い、書き出し時に実ファイル+ファイル内オフセットへ変換する
- **Note のステータス**: `未処理`(貼っただけ)→ `昇格済み`(Selection 化した)/ `破棄`(見返したけどハズレ)。付箋一覧が「見たか見てないか分からないゴミ箱」化するのを防ぐ

## 5. コアワークフロー(2 段階メモ方式 + 進捗管理)

1. **ザッピング(俯瞰)**: サムネイルストリップ+発話区間バーを眺めて、気になる箇所に**付箋メモ**(ワンタップ・1 点打ち)をポンポン貼る。見終わったクリップは**確認済みマーク**を付けて消化を可視化する
2. **詰め(選定)**: 未処理の付箋を順に再生で確認し、良かった箇所は**イン点/アウト点で範囲選定に昇格**、ハズレは**破棄**。付箋の未処理がゼロになったらその Day の選定完了
3. **書き出し**: Selection を並べたラフカットを FCPXML で書き出し、DaVinci Resolve で開いて本編集スタート

数週間にわたる多セッション運用が前提。**アプリを開いた瞬間に「どの Day がどこまで進んでいるか」が見える**ことを最重要の UX とする。

## 6. 機能仕様(フェーズ別)

### Phase 1 — MVP「俯瞰+メモ+持久戦装備」

- [x] **素材取り込み**: 外付けドライブのフォルダを指定してスキャン。撮影日時で日別(Day)に自動振り分け、連番ファイルを論理クリップにグルーピング。ffprobe でメタデータ(解像度・コーデック・長さ・撮影日時)抽出
- [x] **サムネイルストリップ**: クリップごとに一定間隔のサムネイルを ffmpeg で生成し、漫画のコマのように俯瞰表示。**粗→密の 2 パス生成**(まず 60 秒間隔で全体を出す → 裏で 10 秒間隔に詰める)で、初回起動直後から使い始められるようにする
- [x] **発話区間バー(VAD)**: Silero VAD で「喋っている場所」を検出し、ストリップ下に発話バーとして表示。車載映像のように絵が変わらない素材で「会話があった瞬間」を見つける第 2 の俯瞰軸。VAD は CPU でも高速なので文字起こし(Phase 3)とは独立して先行実行する
- [x] **プレーヤー+付箋メモ**: サムネイルクリックでその位置から再生。再生中にショートカット一発で現在位置に付箋。付箋一覧はサイドパネルに表示しクリックでジャンプ
- [x] **レビュー効率の操作系**: 倍速再生(1x / 1.5x / 2x 以上)、±10 秒 / ±1 分スキップ、J/K/L 操作をはじめ**キーボードファースト**で設計。等倍でしか見られないツールでは 80 時間は消化できない
- [x] **レビュー進捗管理**: クリップごとの確認ステータス(未確認 / 確認中 / 確認済み)と再生済み区間の自動トラッキング。Day 一覧・クリップ一覧の双方に消化率を表示し、「どこまで見たか」を一目で分かるようにする
- [x] **日別プロジェクトナビ**: Day 単位で切り替え。Day 内は**全カメラ(メイン+スマホ等)を撮影時刻順に混在表示**し、「その日の流れ」として俯瞰できるようにする。Day ごとの素材本数・総時間・メモ数・消化率を一覧表示
- [x] **バックグラウンドジョブ基盤**: サムネイル生成・VAD は UI をブロックせずに裏で実行し、生成済み分から順次表示する
- [x] **データ永続化**: メモ・選定・解析結果はプロジェクトフォルダ内に JSON で保存(素材ドライブとは分離可能に)。**保存のたびに世代バックアップ**を残す(数十時間分のレビュー成果が入る最重要データのため)。サムネイル等の生成物はキャッシュディレクトリに保存

### Phase 2 — 選定と DaVinci 連携

- [x] **範囲選定(Selection)**: イン点/アウト点を打って範囲を切り出し。メモ・タグ(#絶景 #飯 #トラブル など自由タグ)・★評価を付与
- [x] **付箋 → Selection 昇格フロー**: 付箋一覧から「この付箋を範囲選定に昇格」(付箋位置を起点にイン/アウト点を調整)/「破棄」をワンアクションで。未処理の付箋だけを順に処理していく**トリアージビュー**を用意する
- [x] **再生不可素材のプロキシ生成**: ブラウザで直接再生できない素材(HEVC / 10-bit 等、主にスマホ素材)を検出し、軽量 H.264 プロキシを自動生成。スマホ素材はハイライトの宝庫なので、**レビュー対象から漏れない**ことを保証する(本数・尺が少ないので処理コストは小さい)
- [x] **ラフカット書き出し**: Day 内の Selection を時系列(または手動並べ替え)で並べたタイムラインを **FCPXML** で書き出し。DaVinci Resolve で開くと「使うところだけが並んだ状態」になる。分割ファイル → 実ファイル+オフセットへの変換もここで吸収
- [x] **汎用書き出し**: タイムコード付き一覧の CSV / Markdown 書き出し(保険・メモ共有用)

### Phase 3 — 文字起こしと検索

- [x] **文字起こし**: whisper.cpp(CPU・夜間バッチ)で、Phase 1 の VAD が抽出済みの**発話区間のみ**を処理(車載素材の無音部をスキップして処理量を大幅削減)。日本語+中国語の混在に対応するため言語自動判定を使い、モデルサイズは精度と速度のバランスを見て選定(まず medium 相当を 1 日分で試し、実測で決める)
- [x] **夜間バッチ運用 UI**: 「Day 3 の文字起こしを今夜実行」のように日単位でキューイングし、進捗・完了通知を UI に表示。中断・再開可能
- [x] **横断検索**: 文字起こしテキスト・メモ・タグを全 Day 横断で検索。「あの話してたシーンどこだっけ」をテキストで見つけてジャンプ

### Phase 4 — リッチ化(必要になったら)

- [x] **GPS・地図ビュー**: 素材のメタデータ/GPS ログから撮影地の地図上に素材をマッピング。「あの街で撮った素材」を地図から探せる
- [x] **シーン自動分割**: 映像の場面転換検出でクリップ内にチャプターを自動付与
- [x] **全素材の一括プロキシ化**: Phase 2 の「再生不可素材のみ」を超えて、4K 素材の再生が重い場合などに全素材プロキシ運用へ拡張

## 7. 非機能要件・設計上の注意

- **80 時間規模が前提**: サムネイル生成・VAD・文字起こしは全てバックグラウンドジョブ。UI をブロックしない。生成済み分から順次使える(全部終わるまで待たせない)
- **素材は読み取り専用**: 元ファイルには一切書き込まない。生成物・メモはすべて別ディレクトリ
- **メモデータの保全**: JSON 保存は世代バックアップ付き。プロジェクトフォルダごと git 管理することも推奨(生成物キャッシュは除外)
- **キャッシュサイズの見積もり**: サムネイル(80h × 10 秒間隔 ≒ 約 3 万枚)は低解像度 JPEG で 1GB 前後に収める
- **外付けドライブの抜き差し耐性**: ドライブ未接続でもメモの閲覧・編集はできる(再生・サムネ生成だけ不可)
- **ブラウザ再生の制約**: H.264 MP4 はそのまま再生、それ以外は Phase 2 のプロキシ経由

## 8. 未決事項(実装しながら決める)

- **論理クリップの複数ファイル跨ぎ再生方式**: ブラウザの `<video>` は複数 MP4 をシームレス再生できない。境界でのソース切り替え(一瞬の途切れを許容)/ MSE / 連結プロキシのいずれかを実装時に検証して決める
- 文字起こしモデルのサイズ(small / medium / large-v3-turbo)— 1 日分でベンチマークして決定
- FCPXML の DaVinci 互換性の細部(マーカーの色・メモの載せ方)— 実機で検証
- サムネイル間隔のデフォルト値(密パスを 10 秒 or 30 秒)— 使用感で調整
- Tauri 化のタイミング — Web アプリで不便が出てきたら

## 9. 当面のマイルストーン

1. **M1(MVP)**: 取り込み → サムネイル俯瞰(粗→密)+発話バー → 倍速・キーボード操作付きプレーヤー → 付箋メモ → 進捗管理 → JSON 保存+バックアップ(Phase 1 全部)
2. **M2**: 範囲選定+付箋トリアージ → 再生不可素材のプロキシ → FCPXML ラフカット書き出し → DaVinci で開けることを実機確認
3. **M3**: whisper.cpp 文字起こし(VAD 区間のみ)+夜間バッチ運用+検索
4. **M4**: 地図・シーン分割など、使ってみて欲しくなったもの
