# アプリアイコン制作指示書

Video Edit Helper のアプリアイコンを新規制作するための指示書。
デザイナー / AI 画像生成 / 自作(Figma・Illustrator 等)のいずれでも実行できるよう、**コンセプト → デザイン要件 → 技術仕様 → 制作手順 → チェックリスト**の順でまとめる。

---

## 1. 目的と現状

- 現在 `src-tauri/icons/` に入っているアイコン(ティール×アンバーのリング状)は汎用プレースホルダで、アプリの性格を表現できていない。実質デフォルト状態。
- 本アプリ専用のアイデンティティを持つアイコンに差し替える。
- 成果物は **マスター画像 1 枚(1024×1024 PNG)**。ここから `tauri icon` コマンドで全プラットフォーム用アイコンを自動生成する(手順は §4・§5)。

---

## 2. アプリのコンセプト(デザインの拠り所)

このアイコンで表現したいアプリの本質:

- **正体**: 旅行 VLOG など長時間(数十時間級)撮りためた動画素材を、**俯瞰・メモ・選定**して DaVinci Resolve などの編集ソフトへ渡す「編集補助(ヘルパー)」ツール。編集ソフトそのものではない。
- **中核体験**:
  - 大量素材の**俯瞰**(サムネイルストリップ、日別振り分け)
  - **選定**(`I`/`O` でイン点・アウト点、★評価、「使うところだけ」を抜き出す)
  - **キーボードファースト**(J/K/L、倍速)
  - **文字起こし・横断検索**で「あの話してたシーン」へジャンプ
- **性格のキーワード**: 俯瞰する / 選び抜く / 素早い / 整理する / 編集の"前工程"を助ける相棒。

> デザインの狙い:「膨大な映像の中から使うカットを選び抜く」という**選定**の価値、または「全体を見渡す」**俯瞰**の価値のどちらかを、一目で伝えるモチーフにする。

---

## 3. デザイン要件

### 3.1 モチーフ案(いずれか、または組み合わせ)

推奨順:

1. **フィルムストリップ + 選定マーク(第一推奨)**
   横に並んだフィルムのコマ(サムネイル)の中で、1 コマだけがハイライト/枠取り/チェックされている。「大量の素材から選び抜く」というアプリの核をそのまま表す。
2. **イン点・アウト点ブラケット `[ ]`**
   映像フレームを `[` と `]` のブラケットで挟む。本アプリの選定操作(`I`/`O`)の象徴。ミニマルで小サイズに強い。
3. **俯瞰する視点(グリッド + フォーカス)**
   小さなコマがグリッド状に並び、1 つがフォーカスされている。「俯瞰して見つける」を表現。
4. **再生三角 + フィルム/ストリップの融合**
   映像アプリらしさ(再生三角形)と選定(コマ・チェック)を組み合わせる。

避けたいもの:

- DaVinci Resolve 等、他社編集ソフトのアイコンと紛らわしい意匠。
- クラッパーボード(カチンコ)や単なる再生ボタンだけ → 「撮影/プレーヤー」に見え、"選定ヘルパー"という差別化が伝わらない。
- 現行のリング/無限マーク意匠の踏襲(汎用的すぎる)。

### 3.2 カラーパレット

プロジェクト内に既存の色資産が 2 系統ある。**アプリ起動画面(スプラッシュ)の配色に寄せる**ことを推奨(実際のアプリ第一印象と揃うため)。

| 用途 | 推奨 HEX | 出典 |
|---|---|---|
| ベース背景(ダーク) | `#15171C` | スプラッシュ背景 |
| プライマリ(青) | `#4AA3FF` | スプラッシュのアクティブ色 |
| アクセント(緑=確定/選定) | `#57C97A` | スプラッシュの完了色 |
| 明色(前景/白) | `#E6E8EC` | スプラッシュ前景 |

代替として、現行アイコンのティール×アンバー系(`#2BC4DE` / `#FFBC15` 付近)を活かす方向も可。ただし採用する場合もダークベースと組み合わせて奥行きを出すこと。

指針:

- **2〜3 色まで**に絞る。多色はアイコンサイズでは濁る。
- ダークベース + 明るいアクセント 1 色(選定を表す色)を核にすると、「選び抜く 1 カット」を色でも強調できる。
- 背景は単色 or ごく浅いグラデーション(上→下で 5〜10% 程度)まで。派手なグラデ・光沢は避ける。

### 3.3 スタイル / トーン

- **フラット寄り + 最小限の奥行き**。macOS Sonoma/Sequoia 以降のモダンなアプリアイコンのトーン。過度なスキューモーフィズム(革・金属・強い影)は避ける。
- 形状は幾何学的でクリーン。線幅は太めに(小サイズで消えないように)。
- 情報量は少なく。**要素は 1〜2 個**に絞り、16px でも要素が識別できること。

### 3.4 小サイズでの可読性(最重要)

アイコンは 16px 相当(メニューバー・タスクバー・Finder リスト表示)でも表示される。

- 32px・16px に縮小してディテールが潰れないか必ず確認する。
- 細い線・小さな文字・微妙な色差は禁止。
- モチーフのシルエットだけで意味が伝わることを目標に。

### 3.5 macOS 角丸・セーフエリアの注意(必読)

> **重要**: `tauri icon` は入力画像を**そのまま縮小するだけ**で、macOS 風の角丸(スーパー楕円)マスクや余白は**自動付与しない**。macOS ネイティブな見た目にしたい場合、マスター画像に角丸と余白を**焼き込む**必要がある。

macOS 標準アプリに馴染ませる場合のレイアウト目安(1024×1024 キャンバス):

- キャンバス: 1024×1024、**背景は透過(アルファ付き PNG)**。
- アイコン本体(角丸の四角形 = スーパー楕円): おおむね **824〜880px 角**、中央配置、周囲に **約 72〜100px の透過マージン**。
- 角丸半径: 本体の一辺の **約 22.4%**(例: 本体 824px なら半径 ≈ 185px)。
- 主要モチーフはさらに内側の**セーフゾーン(中央 ~70%)**に収める(縮小・マスクで欠けないため)。

Windows(`.ico` / Square*Logo)は角丸なしの矩形で表示されるため、本体が角丸でも破綻しないデザインにする(モチーフが四隅に張り出さない)。

## 4. 技術仕様(生成の仕組み)

### 4.1 納品物 = マスター画像 1 枚

| 項目 | 仕様 |
|---|---|
| ファイル | `app-icon.png`(名前は任意。§5 のコマンドで指定) |
| サイズ | **1024×1024 px** |
| 形式 | PNG、**アルファチャンネル(透過)あり** |
| カラー | sRGB |
| 中身 | §3.5 のレイアウトに従い、角丸・余白を焼き込んだ状態 |

ベクター(SVG / Illustrator / Figma)で作成し、最終書き出しで 1024px PNG にするのが理想(将来の再出力が容易)。作業用ソースも一緒に納品する(`docs/assets/` 等に保存)。

### 4.2 `tauri icon` が生成するファイル(差し替え対象)

マスター 1 枚から下記が `src-tauri/icons/` に生成される。個別に手作業で用意する必要はない。

- `32x32.png`, `128x128.png`, `128x128@2x.png`, `icon.png`
- `icon.icns`(macOS)
- `icon.ico`(Windows)
- `StoreLogo.png`, `Square30x30Logo.png` 〜 `Square310x310Logo.png`(Windows Store/MSIX 各サイズ)

`tauri.conf.json` の `bundle.icon` は既にこれらを参照しているため、**設定変更は不要**。

## 5. 制作フロー(手順)

1. **デザイン**: §2・§3 に沿ってマスター画像を制作 → `1024×1024` の透過 PNG で書き出す(例: `docs/assets/app-icon.png`)。
2. **全アイコン生成**: リポジトリルートで実行。
   ```bash
   pnpm tauri icon docs/assets/app-icon.png
   ```
   → `src-tauri/icons/` 内の全ファイルが上書き生成される。
3. **確認**:
   ```bash
   pnpm tauri:dev
   ```
   でウィンドウ / Dock / タスクバーのアイコンを確認。ビルド版は `pnpm tauri:build` で生成物(`.app` / インストーラ)を確認。
4. **小サイズ確認**: `src-tauri/icons/32x32.png` を実寸で確認し、潰れがないかチェック(§3.4)。
5. **コミット**: 生成された `src-tauri/icons/*` とマスター画像(`docs/assets/`)を含めてコミットする。

> 注: `pnpm tauri:build` 時は `beforeBuildCommand`(`tauri:prepare`)でフロントエンドやバイナリの準備が走る。アイコン生成(手順 2)は事前に済ませておけばよく、ビルドのたびに毎回実行する必要はない。

## 6. AI 画像生成を使う場合(任意)

画像生成 AI に描かせる場合のプロンプト例(英語)。生成後は §5 の手順で組み込み、§3.4/§3.5 を満たすよう調整・トリミングする。

```
A modern macOS app icon for a video editing helper tool.
A dark rounded-square (superellipse) app tile on a transparent background,
deep charcoal base (#15171C) with a subtle top-to-bottom gradient.
Centered motif: a horizontal filmstrip of small video frames where exactly
one frame is highlighted / selected with a bright blue-to-green accent
(#4AA3FF to #57C97A) and a check mark, conveying "picking the best cut
from lots of footage". Flat, clean, geometric, minimal, bold shapes,
2-3 colors only, high legibility at small sizes, generous safe-area
padding, no text, no logos. 1024x1024.
```

代替モチーフ(ブラケット案)にする場合は、`filmstrip ... one frame highlighted` の部分を
`a single video frame framed by bright in/out brackets "[" and "]"` に置き換える。

## 7. 完成チェックリスト

- [ ] マスターは 1024×1024・透過 PNG・sRGB
- [ ] モチーフが「選定 / 俯瞰」というアプリの核を一目で伝える
- [ ] 色は 2〜3 色、ダークベース + アクセントで統一感がある
- [ ] macOS 角丸・余白・セーフゾーンを焼き込み済み(§3.5)
- [ ] 四隅にモチーフが張り出しておらず、Windows の矩形表示でも破綻しない
- [ ] 32px / 16px 相当でも要素が識別でき、線が消えない
- [ ] `pnpm tauri icon <master.png>` で全ファイルを再生成した
- [ ] `pnpm tauri:dev` でウィンドウ / Dock / タスクバー表示を確認した
- [ ] 生成物(`src-tauri/icons/*`)とソースをコミットした
- [ ] 他社編集ソフトのアイコンと紛らわしくない
